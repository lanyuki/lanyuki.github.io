采样/选词策略

发表于 2025-01-15 分类于 AI-learning Valine：

在语言模型生成概率分布后，采取哪种策略来选择下一个token,显得极其重要。常见的策略有：

Greedy Search (Maximization)

Beam Search

Temperature Sampling

Top-K Sampling

Top-P Sampling (Nucleus sampling)

Greedy Search解码（贪婪）

    Greedy search方法的思想较为简单，就是直接选择概率分布中概率最大的token作为当前解码出来的词。 但是，该方法的问题在于，如果我们总是选择概率最大的词，将会生成很多重复的句子，此方法也是基于统计学的方法。

Beam Search 解码

    另一种比较流行的解码方法叫beam search，该方法是对greedy search的扩展版本，返回一系列最有可能的输出序列。

    和greedy search选择可能性最大的构成序列不同，beam search在n

步时，生成n+1步的所有可能组成，并从中选择k个概率最大的组合，其中k为指定的搜索参数。

    我们在开始的位置不用随机选择，而是选择K个最可能在开始位置的词语作为序列的第一个词。

    当K取1时，即为Greedy Search，而在大多数机器翻译的任务中 K一般取值5-10。当K较大时，往往会带来较好的结果，因为保留更多的选择性，更可能带来最佳的组合，相应的，也会增加计算成本和解码速度。

    举例说明上述表述。首先，定义一个函数，该函数在给定一个序列（假设长度为N, 词汇表长度为V）的概率分布(矩阵，N x V)，以及搜索参数K时，得到解码结果。在每一个step，每一个候选子序列( candidate sequence)扩展所有可能的下一个子token，然后按照score进行排序，并选择score最大的K个子序列，作为当前step的解码结果。重复上述过程，直到迭代结束。

    一般来说，概率值时较小的数值，经过一些列连乘后，会更小，为防止下溢（underflowing the floating point numbers),将其计算转换为取其对数，然后相加的过程。

TEMPERATURE 采样

    Temperature sampling的想法源于热力统计学的概念，貌似是跟熵有一些关系具体没有过多了解，主要就是温度高往往意味着更容易是低能量状态。在概率模型中， logits 代表能量值，将其送入softmax函数钱，除以temperature值，得到最终的采样概率分布。

TOP-K采样

TOP-K也是基于概率的生成，主要思路个人理解是二次贪婪，可以理解为是在第一次的时候贪婪不彻底，选出k个（多个），然后在将这k个token重新进行取概率，然后再选择概率大的，选出该词作为输出。

TOP-P采样

top-p的方式与之前的采样方式都有所不同，之前可以理解为就是低概率以及中概率的词都直接不要，但是top-p选择token时是只要这个词的累计概率超过一个阈值p，就加入备选，这样使得中低概率的词也能被选进来，增加中小概率词的生成概率。

最后再说说个人对于这些选词方法的“偏门”的理解，就是这种选词方法不仅是在单纯的生成当中可能会使用，在于一些大模型安全领域也有一些使用。特别是在某些黑盒模型的文本生成检测当中，由于模型大概率也是使用了上述的选token方法，所以可以采用在多次的同样的文本生成时，统计一些模型的特征词汇出现的频率，来判断是否是由ai生成的文本，也算是新时代的图灵测试。

VIT-BART-T5-yolos原理简述

发表于 2025-01-15 分类于 AI-learning Valine：
Vit模型，把一个个小图片过一个线性投影层，原来的图片像素点拉直（比如是一个128\*20\*20的图像，变成一个128\*400），乘上那个线性投影层层（w）变成一个个向量(粉色的)（即原来的128\*400现在变成了一个128\*1的向量）。模仿nlp,在每个位置都加一个位置向量，紫色加粉色就是一个位置加图像，两个进行位加，这个就代表这个图片的整体信息。最前面加一个cls，代表一个图片，然后采用mlp进行识别。 这个yolos模型，左侧相当于一个vit模型，将所有图片方面的信息转化成向量，右侧则是一些候选目标（向量），通过transformer结构进行解码，最后输出置信度，类别，位置。

这整个模型就类似T5-BART模型

引申T5-BART

这个是bert模型，在BERT源码中，在预测位置可以使用哪些信息是由一个称为attention\_mask的参数来控制的。BERT最重要的预训练任务是预测masked token，并使用整个输入来获取更完全的信息以进行更准确的预测。这对于那些允许利用位置之后的信息来预测位置的任务是有效的，但是对于诸如文本生成之类的任务则没有多大用处，这些对位置的预测只能取决于先前生成的单词。 这个是GPT模型。GPT预训练任务使用的是autoregressive的思想，使用已经解码出的信息来预测下一个位置。该种模式对于生成任务更为有效，而对于那些可以使用全局输入来得到输出的下游任务则比较差劲。 这个则是BART模型，主要就是把BERT和GPT模型组在了一起。这么看，那个yolos和BART模型确实有点像。 Clip模型，如图，图上信息很多。此时正负样本的数量不一样，此时的负样本需要采样N个出来，所采用的策略是采取与正样本相似度较高的负样本，就是内积趋向为1的负样本（相似度低说明区分得很好，没有必要再去区分。而这里的相似度是采用余弦做的） 上面一步，生成一个图像的压缩的token：

（ITC (Image-Text Contrastive
Learning)：目的：使模型能够学习图像和文本之间的对齐关系，促进模型理解哪些图像和哪些文本是匹配的。实现：通过对比损失（contrastive
loss）来训练模型，使得匹配的图像-文本对的特征在嵌入空间中距离更近，而不匹配的对的特征距离更远。通过使用一个图像编码器和一个文本编码器将图像和文本分别编码为向量，然后计算它们之间的相似度。损失函数：对比损失函数通常采用交叉熵损失）

（ITM (Image-Text
Matching)：目的：训练模型判断一对图像和文本是否匹配。实现：在图像和文本特征的基础上添加一个分类头，用于二分类任务（匹配和不匹配）。模型需要预测给定的图像和文本是否是匹配的损失函数：使用二分类交叉熵损失）

LM (Language
Modeling)：目的：训练模型生成描述图像的文本，提升模型的文本生成能力。实现：使用语言模型的损失函数来优化模型，使得模型能够根据图像生成自然语言描述。这通常包括使用一个语言模型（如GPT）来生成文本，并计算生成文本与参考文本之间的损失。损失函数：使用自回归语言建模的交叉熵损失

ITC：通过对比学习，使模型能够在嵌入空间中对齐图像和文本。

ITM：通过匹配任务，增强模型判断图像和文本是否匹配的能 力。

LM：通过语言建模任务，提升模型生成文本描述的能力
